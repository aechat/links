# Скрипт `extractMarkedText`

Файл: `scripts/extractMarkedText.js`

## Назначение

Диагностически извлекает короткий текст из тегов с `className` или `class` в статьях. Нужен, чтобы быстро посмотреть, какие помеченные фрагменты реально используются в `mark`, `span` и похожих элементах.

## Запуск

```bash
yarn extract:marked-text
```

или напрямую:

```bash
node scripts/extractMarkedText.js
```

## Как работает

1. Рекурсивно обходит `src/pages/sections`.
2. Берёт файлы `.tsx`, `.ts`, `.jsx`, `.js`.
3. Находит теги, где есть `className` или `class`.
4. Извлекает короткий внутренний текст:
   - не больше `280` символов;
   - не больше двух вложенных тегов.
5. Группирует результат по файлам и по парам `tag.class`.

## Результат

Скрипт пишет отчёт в новую директорию:

```text
scripts/logs/mark-extract-<timestamp>/
```

Внутри создаются:

- `index.log` — общий индекс;
- `by-file/*.log` — группировка по исходным файлам;
- `by-class/*.log` — группировка по `tag.class`.

## Важный нюанс

Скрипт не исправляет файлы и не является линтером. Он создаёт диагностические логи, поэтому перед коммитом проверьте, что отчёты из `scripts/logs` не попали в индекс случайно.
