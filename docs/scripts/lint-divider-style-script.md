# Скрипт `lintDividerStyle`

Файл: `scripts/lintDividerStyle.js`

## Назначение

Проверяет тексты `<Divider>...</Divider>` в статьях, чтобы заголовки внутренних блоков оставались единообразными.

## Запуск

```bash
yarn lint:divider-style
```

или напрямую:

```bash
node scripts/lintDividerStyle.js
```

Строгий режим:

```bash
node scripts/lintDividerStyle.js --strict
```

## Что проверяет

1. Обходит `src/pages/sections`.
2. Проверяет файлы `.tsx`, `.ts`, `.jsx`, `.js`.
3. Ищет блоки `<Divider>...</Divider>`.
4. Предупреждает, если текст `Divider` содержит вопросительный знак.
5. Завершает работу с ошибкой, если известное название приложения или плагина в `Divider` не обёрнуто в нужный `mark`:
   - приложения — `<mark className="app">...</mark>`;
   - плагины — `<mark className="plugin">...</mark>`.

В режиме `--strict` вопросительный знак в `Divider` тоже становится ошибкой.

## Когда использовать

- перед правками массовой структуры статей;
- после добавления новых `Divider`;
- перед коммитом с изменениями контента, где есть названия приложений или плагинов внутри `Divider`.

## Ограничение

Скрипт проверяет только список известных названий внутри самого файла. Если появляется новый часто используемый инструмент, его нужно добавить в `APP_NAMES` или `PLUGIN_NAMES`.
