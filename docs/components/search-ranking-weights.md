# Веса ранжирования поиска

Файл: `src/components/features/searchEngine/searchScoringUtilities.ts`

## Назначение

Константы в `searchScoringUtilities.ts` управляют балансом:

- точность выдачи (что должно быть выше),
- устойчивость к опечаткам,
- производительность при большом количестве кандидатов.

## `SEARCH_SCORING_WEIGHTS`

- `tagAndTitleMatch`: базовый бонус, когда совпадения есть одновременно в заголовке и тегах.
- `tagOrTitleMatch`: базовый бонус, когда совпадения есть только в одном из полей (`title` или `tag`).
- `titleAndTagMatchCountStep`: шаговой бонус за число совпавших слов в `title/tag`.
- `contentMatchCount`: бонус за число совпадений в контенте.
- `entityMatchCount`: бонус за число совпадений в `entity` (ссылки, выделенные сущности).
- `phraseTitle`: большой бонус за фразовое совпадение запроса в заголовке.
- `phraseTag`: бонус за фразовое совпадение в тегах (с учетом демпфирования длинных тегов).
- `contentPhraseMatch`: бонус за фразовое совпадение в контенте.
- `foldedTitleFallback`: компенсационный бонус, если в заголовке есть сильные не-exact сигналы (`folded/stem/consonant/prefix`) при отсутствии фразового попадания.
- `contentOnlyPenalty`: штраф для результатов, где сигнал только из контента без `title/tag/entity`.

## `SEARCH_FIELD_WEIGHTS`

Вес полей для match-type скоринга (`exact/prefix/folded/consonant/stem`):

- `title`: максимальный приоритет.
- `tag`: высокий приоритет.
- `entity`: высокий приоритет, но ниже заголовка.
- `content`: минимальный приоритет.

## `SEARCH_PROXIMITY_WEIGHTS`

Коэффициенты бонуса близости слов запроса (чем слова запроса ближе друг к другу в поле, тем выше):

- `title` > `tag` > `entity` > `content`.

## `SEARCH_LEADING_WEIGHTS`

Коэффициенты бонуса ранней позиции первого слова запроса:

- самый высокий вес у `title`,
- самый низкий у `content`.

## `SEARCH_ORDER_WEIGHTS`

Коэффициенты бонуса за сохранение порядка слов запроса:

- `title` получает максимальный бонус,
- `content` — минимальный.

## `SEARCH_RESCORE_CONFIG`

- `activationThreshold`: порог количества кандидатов, после которого включается ограниченный полный рескоринг.
- `candidateLimit`: сколько лучших кандидатов (по предварительному скору) отправляется в полный тяжелый рескоринг.

Смысл: при очень больших выборках снижать CPU-нагрузку без заметной потери качества на верхних позициях.

## Дополнительные константы

- `WORD_MATCH_SCORE` задаёт вес типов совпадений: `exact`, `prefix`, `folded`, `consonant`, `stem`.
- `SHORT_QUERY_*` задают корректировки для коротких запросов длиной до 3 символов.
- `PROXIMITY_*`, `ORDER_*`, `LEADING_*` задают базовые лимиты и decay для бонусов близости, порядка и ранней позиции.
