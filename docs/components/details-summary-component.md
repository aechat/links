# Компонент `DetailsSummary`

Файл реализации: `src/components/content/DetailsSummary.tsx`.

---

## Назначение

- показывает заголовок вопроса и раскрываемый контент;
- нумерует спойлеры внутри статьи и добавляет номер в заголовок;
- работает с hash-навигацией и ссылками на конкретный ответ;
- управляет затемнением фона при открытом спойлере;
- обрабатывает внутренние и внешние ссылки, а также копирование ссылок;
- поддерживает long-press-копирование ссылок из блоков `.flexible-links`.

Компонент рассчитан на использование внутри контейнера `.article-content`.

---

## Интерфейс

Упрощённый интерфейс пропсов:

```ts
interface DetailsSummaryProperties {
  anchor: string;
  children: React.ReactNode;
  tag?: string;
  title: string;
}
```

- `title` — заголовок вопроса;
- `anchor` — текстовый идентификатор для ссылок;
- `tag` — строка с тегами через запятую и пробел;
- `children` — JSX-содержимое ответа.

Заголовок рендерится внутри `<h2>` и дополняется номером, который генерируется автоматически, например `1.3. ...`.

---

## Базовый пример

```tsx
import DetailsSummary from "../../../components/content/DetailsSummary";

<DetailsSummary
  anchor="home-screen"
  tag="стартовая страница, домашний экран, home screen"
  title="Почему не работает окно с недавними проектами и как его отключить?"
>
  <p>Короткое объяснение проблемы и несколько шагов решения.</p>
</DetailsSummary>;
```

Теги передавайте списком через запятую и пробел, чтобы компонент корректно разделял их на элементы.

## Якоря и ссылки

`DetailsSummary` работает и с текстовыми, и с числовыми якорями.

- Текстовый якорь задаётся в `anchor`:

  ```tsx
  <DetailsSummary
    anchor="switch-language"
    tag="изменить язык, смена языка, установка английской версии"
    title="Почему мне советуют поставить английский язык программы и как это сделать?"
  >
    {/* ответ */}
  </DetailsSummary>
  ```

- Ссылка из другого места:

  ```tsx
  <a href="#switch-language">как изменить язык программы</a>
  ```

При открытии страницы с таким hash компонент найдёт и раскроет нужный спойлер.

Кнопка копирования в заголовке формирует URL с `anchor`, например:

```text
https://aechat.ru/aefaq#switch-language
```

При клике на ссылки внутри ответа:

- внутренние ссылки вида `#anchor`, ведущие к другому спойлеру, открываются через модальное подтверждение перехода;
- внешние `http/https`-ссылки также показывают подтверждение перед открытием.

## Нумерация и прокрутка

Внутри `.article-content` компонент:

- присваивает каждому спойлеру числовой id в формате `<номер-блока>.<номер-спойлера>`;
- добавляет этот номер в начало заголовка;
- при открытии выполняет плавную прокрутку с учётом высоты шапки;
- синхронизирует hash в URL.

Если спойлер открыт по ссылке, компонент раскрывает его и прокручивает страницу к заголовку.

## Теги вопросов

Свойство `tag` принимает строку с тегами, разделёнными запятыми:

```tsx
tag = "helper.exe не найден, выключить защитник, выключить антивирус";
```

Поведение:

- строка делится на отдельные теги;
- если тегов много, в свёрнутом состоянии показывается случайная подвыборка тегов (до лимита), а остальные скрываются под кнопкой «и ещё N тегов»;
- по клику список разворачивается полностью.

Теги используются в поиске и помогают быстрее находить нужный вопрос.

## Вложенные спойлеры

Внутри `DetailsSummary` можно использовать `NestedDetailsSummary` для длинных ответов:

```tsx
import NestedDetailsSummary from "../../../components/content/NestedDetailsSummary";

<DetailsSummary
  anchor="get-started"
  tag="для новичка, курсы, уроки"
  title="Я первый раз открыл программу. С чего начать обучение?"
>
  <NestedDetailsSummary title="Курсы для начинающих">
    {/* подробный список ресурсов */}
  </NestedDetailsSummary>
</DetailsSummary>;
```

Подробнее — в [`nested-details-summary-component.md`](./nested-details-summary-component.md).
