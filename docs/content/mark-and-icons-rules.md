# Правила для `mark` и семантических классов

Стили: `src/styles/components/_mark.scss`.

## Зачем использовать `mark`

- выделение сущностей, которые проще читать с визуальным маркером;
- единый стиль для файлов, путей, приложений, кнопок и терминов.

## Основные классы

- `file`, `path`, `copy`;
- `app`, `plugin`, `company`, `user`;
- `web`;
- `select`, `key`, `tag`, `word`, `quote`, `code`, `file-code`, `version`;
- `hardware`;
- `codec-param`;
- `term`;
- `color-space`;
- `image`, `video`, `audio`;
- `info`, `warning`, `danger`.

## Практика

- выбирайте класс по смыслу, а не по «красивой» иконке;
- не делайте длинные абзацы из сплошных `mark`;
- для обычного текста оставляйте обычный `p`;
- названия элементов интерфейса (поля, кнопки, пункты меню, параметры) оформляйте как `mark.select` и с `« »`;
- значения, которые нужно ввести/вставить/указать вручную (`0`, `23`, `320`, `70`), оформляйте как `mark.copy` без кавычек;
- если единица измерения/суффикс задаётся самим интерфейсом (например `Кбит/с`, `МБ`, `%`), выносите её за пределы `mark.copy`;
- отображаемые версии (релизы и диапазоны в тексте) оформляйте через `mark.version`;
- диапазон версий с названием программы оформляйте в виде `от ... до ...` с `mark.app`;
- версии и диапазоны без названия программы пишите обычным текстом, без `mark` (например: `23.6 (2023)`, `от 23.1 до 23.5 (2023)`);
- для `app/plugin` оборачивайте в `mark` только название сущности, а версию рядом оформляйте через `mark.version`;
- для `term/web/company` при конструкции `название + версия` придерживайтесь того же подхода: в `mark` только название, версия снаружи;
- в таблицах перевода/каталогах эффектов (`*WhereFind.tsx`) не дробите устоявшиеся названия вида `Keylight (1.2)` и `CC Light Burst 2.5` без отдельной задачи;
- исключения: устоявшиеся токены и плейсхолдеры (`CC0`, `20XX`) можно оставлять целиком в `mark`.
- формат версий, словарь именований и стиль `Divider` см. в [`version-format-rules.md`](./version-format-rules.md);
- для названий приложений используйте `mark.app` только в контенте статьи;
- не вставляйте `mark` в `title`/`caption` и в текст ссылок `<a>...</a>`.

## Пример

```tsx
<p>
  Откройте <mark className="path">Edit -> Preferences -> Media & Disk Cache</mark> и
  очистите <mark className="file">Disk Cache</mark>.
</p>
<p>
  Для выражения используйте <mark className="code">wiggle(3, 20)</mark>.
</p>
<p>
  Ресурс: <mark className="web">Envato Elements</mark>.
</p>
<p>
  Параметр кодирования: <mark className="codec-param">CBR</mark>.
</p>
<p>
  Язык: <mark className="file-code">JavaScript</mark>.
</p>
<p>
  Компонент: <mark className="hardware">Intel Core</mark>.
</p>
```
