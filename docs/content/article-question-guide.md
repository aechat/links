# Гайд по структуре вопросов в статьях

## Обязательные элементы `DetailsSummary`

- `anchor` обязателен и должен быть стабильным, если на ответ будут ссылаться;
- `title` формулируется как реальная задача пользователя;
- `tag` добавляйте по необходимости, через `", "` (запятая + пробел).

## Правила для `anchor`

- используйте `kebab-case`, например: `media-cache-error`;
- не меняйте `anchor` без необходимости, чтобы не ломать старые ссылки;
- избегайте общих значений вида `test`, `new`, `example`;
- если тема переехала, оставляйте старый `anchor` на наиболее близком ответе.

## Правила для `title`

- заголовок как вопрос пользователя, а не как название раздела;
- один заголовок = одна проблема;
- без лишней вводной части.

Плохо: `Настройки и разные нюансы`. Хорошо: `Почему не открывается окно импорта и как восстановить его?`.

## Правила для `tag`

- теги нужны для поиска, а не для украшения;
- добавляйте синонимы и частые формулировки из реальных запросов;
- не повторяйте слово из `title` в каждом теге.

## Пример

```tsx
<DetailsSummary
  anchor="media-cache-error"
  tag="ошибка кэша, media cache, поврежденный кэш"
  title="Почему появляется какая-то ошибка и как ее исправить?"
>
  <p>...</p>
</DetailsSummary>
```
