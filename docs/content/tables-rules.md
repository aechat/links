# Правила для таблиц

Таблицы используйте только там, где действительно нужны сравнения и матрицы параметров.

## Когда таблица уместна

- сравнение форматов/настроек;
- соответствие сочетаний клавиш и действий;
- матрица «условие -> результат».

## Правила

- заголовки столбцов короткие и однозначные;
- не дублируйте в таблицу то, что проще списком;
- если строк много, добавьте короткое пояснение перед таблицей;
- избегайте «визуальных» таблиц ради оформления.

## Пример использования

```tsx
<table>
  <thead>
    <tr>
      <th>Формат</th>
      <th>Когда использовать</th>
      <th>Ограничения</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>H.264</td>
      <td>Быстрый экспорт предпросмотра</td>
      <td>Потери при повторном рендере</td>
    </tr>
    <tr>
      <td>Apple ProRes</td>
      <td>Промежуточный мастер-файл</td>
      <td>Большой размер файла</td>
    </tr>
  </tbody>
</table>
```
