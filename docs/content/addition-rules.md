# Правила для `Addition`

`Addition` используется для заметных уточнений внутри ответа.

## Типы

- `info` — справка, контекст, полезный нюанс;
- `warning` — важное ограничение или риск;
- `danger` — критичное предупреждение;
- `tldr` — краткий итог длинного ответа.

## Когда использовать

- шаг легко пропустить, но это влияет на результат;
- есть риск потери данных/ошибочных действий;
- нужно отделить «главную мысль» от деталей.

## Когда не использовать

- для обычного текста, который можно оставить абзацем;
- слишком часто подряд, без реальной смысловой нагрузки;
- несколько однотипных уточнений лучше объединять в один блок списком через `<ul>` и `<li>`.

## Чеклист

- выбран корректный `type` по уровню риска (`info`, `warning`, `danger`, `tldr`);
- блок не дублирует обычный абзац и действительно выделяет важное;
- рядом с `Addition` есть основной контекст, а не только предупреждение.

## Антипаттерны

- использовать `danger` для незначительных ремарок;
- ставить несколько `Addition` подряд без нового смысла;
- переносить в `Addition` почти весь текст ответа.

## Пример использования

```tsx
<Addition type="warning">
  <p>Перед очисткой кэша закройте проект и сохраните копию рабочей папки.</p>
</Addition>
```
