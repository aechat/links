# Правила для абзацев (`p`)

Абзац в FAQ-ответах должен быть коротким и целевым.

## Правила

- первый абзац объясняет причину/контекст проблемы;
- следующие абзацы описывают шаги и результат;
- один абзац = одна мысль;
- лучше 2–4 коротких абзаца, чем один длинный блок текста.

## Практика

- пишите конкретно: что делать, где нажать, какой эффект ожидается;
- если шаг потенциально опасный или необратимый, оформите его через `Addition`;
- не перегружайте абзац множеством `mark` подряд.

## Пример использования

```tsx
<p>Ошибка обычно появляется после переноса проекта с другого компьютера из-за сбитых путей к кэшу.</p>
<p>Откройте настройки кэша, укажите новую папку и перезапустите программу.</p>
```
