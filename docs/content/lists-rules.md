# Правила для списков (`ul/ol/li`)

Списки — основной паттерн для инструкций.

## Правила

- каждый `li` должен содержать законченную мысль;
- дополнительные нюансы внутри пункта выносите в `Addition`;
- не создавайте глубокую вложенность без необходимости;
- избегайте пунктов из 1–2 слов без пояснения.

## Практика

- если пунктов много и они тематически разные, разбейте блок `Divider`;
- если список превращается в отдельную подтему, лучше оформите её через `NestedDetailsSummary`.

## Пример использования

```tsx
<ul>
  <li>Проверьте путь к Media Cache в настройках программы.</li>
  <li>Очистите кэш и перезапустите приложение.</li>
  <li>Повторите импорт и проверьте результат.</li>
</ul>
```
