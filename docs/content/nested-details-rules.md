# Правила для `NestedDetailsSummary`

Файл компонента: `src/components/content/NestedDetailsSummary.tsx`.

## Когда использовать

- длинный ответ внутри `DetailsSummary` разбит на самостоятельные подтемы;
- нужно сгруппировать несколько крупных подпунктов без создания нового верхнего спойлера.

## Важные ограничения

- Компонент используется только внутри `DetailsSummary`.
- Вне `DetailsSummary` компонент логирует ошибку в консоль и не рендерится.
- Глубокая вложенность ухудшает читаемость, обычно достаточно одного уровня.

## Пропсы

- `title` — заголовок вложенного блока.
- `children` — содержимое.
- `startOpen` — открыт ли блок по умолчанию.
- `modifierClass` — дополнительный CSS-класс.

## Практика

- Ставьте короткие и предметные заголовки.
- Не дробите ответ на слишком мелкие блоки.
- Если вложенных блоков становится слишком много, лучше создать отдельный `DetailsSummary`.

## Чеклист

- вложенный блок действительно раскрывает подтему, а не повторяет основной текст;
- заголовки вложенных секций короткие и различимые;
- вложенность ограничена одним уровнем, если нет явной необходимости глубже.

## Антипаттерны

- превращать весь ответ в цепочку из вложенных спойлеров;
- использовать `NestedDetailsSummary` вне `DetailsSummary`;
- выносить в вложенные блоки отдельные проблемы, которые лучше оформить как самостоятельные вопросы.

## Пример

```tsx
import NestedDetailsSummary from "../../../components/content/NestedDetailsSummary";

<NestedDetailsSummary title="Проверяем настройки кеша">
  <p>Откройте Preferences и проверьте путь к Disk Cache.</p>
</NestedDetailsSummary>;
```
