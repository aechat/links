# Гайд по структуре ответов в статьях

## Рекомендуемый порядок

1. Коротко: что происходит и почему.
2. Пошаговое решение (`ul/ol` или последовательность абзацев).
3. Ограничения и риски через `Addition`.
4. Проверка результата и следующий шаг, если решение не помогло.

## Если материал длинный

- делите ответ через `Divider`;
- используйте `NestedDetailsSummary` для подтем;
- не смешивайте в одном спойлере несколько разных проблем.

## Компоненты вместо ручных паттернов

- `Addition` для предупреждений и выносок;
- `ArticleMedia` для изображений и видео;
- `ContentFilter` для различий Windows/macOS;
- `CodeSnippet` для кода и команд;
- `HostsAdobe` (`HostsAdobeModal`) для блоков `hosts`.

## Тон и подача

- обращение на «вы»;
- короткие абзацы и конкретные формулировки;
- минимум сленга и воды;
- рядом со ссылкой указывайте, что находится по ней.

## Рабочий каркас ответа

```tsx
<DetailsSummary
  anchor="render-queue-stuck"
  title="Почему зависает очередь рендера и как это исправить?"
>
  <p>Коротко объясните причину и когда это обычно случается.</p>
  <ul>
    <li>Первый шаг.</li>
    <li>Второй шаг.</li>
    <li>Третий шаг с ожидаемым результатом.</li>
  </ul>
  <Addition type="warning">
    <p>Опасный или необратимый шаг выделяйте отдельно.</p>
  </Addition>
</DetailsSummary>
```

## Частые ошибки

- длинный монолитный текст без структуры;
- шаги без ожидаемого результата;
- предупреждения спрятаны внутри обычного абзаца;
- в одном ответе одновременно разобраны разные симптомы.
