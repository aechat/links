# Правила для `ContentFilter`

Файл компонента: `src/components/content/ContentFilter.tsx`.

## Когда использовать

- один и тот же сценарий отличается для Windows и macOS;
- нужно держать оба варианта в одном `DetailsSummary`.

## Как работает

- Компонент автоматически определяет платформу пользователя.
- `iOS` трактуется как `macOS`-ветка.
- Если переданы оба варианта (`windowsContent` и `macContent`), показывается переключатель.
- Если передан только один вариант, переключатель скрывается.

## Практика

- Держите структуру шагов в обеих ветках максимально одинаковой.
- Давайте общие пояснения до `ContentFilter`, а различия оставляйте внутри веток.
- Не используйте `ContentFilter`, если различий по платформам нет.

## Чеклист

- в ветках `windowsContent` и `macContent` описан один и тот же сценарий;
- отличия действительно зависят от платформы;
- общие шаги вынесены вне `ContentFilter`, чтобы не дублировать текст.

## Антипаттерны

- использовать `ContentFilter` при отсутствии реальных различий между ОС;
- делать разную логику в ветках без пояснения причин;
- смешивать платформенные различия с не связанными подтемами.

## Пример

```tsx
import ContentFilter from "../../../components/content/ContentFilter";

<ContentFilter
  windowsContent={
    <p>
      Чтобы изменить настройки внешнего вида программы, перейдите в{" "}
      <mark className="select">«Edit» → «Preferences» → «Appearance»</mark>.
    </p>
  }
  macContent={
    <p>
      Чтобы изменить настройки внешнего вида программы, перейдите в{" "}
      <mark className="select">«After Effects» → «Preferences» → «Appearance»</mark>.
    </p>
  }
/>;
```
