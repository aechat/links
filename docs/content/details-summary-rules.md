# Правила для `DetailsSummary`

Файл компонента: `src/components/content/DetailsSummary.tsx`.

## Правила использования

- один `DetailsSummary` = один конкретный пользовательский сценарий;
- `anchor` задавайте в `kebab-case` и не меняйте без необходимости;
- `title` формулируйте как понятный вопрос пользователя;
- `tag` добавляйте только для поисковых синонимов и уточнений.

## Чеклист

- у каждого блока заполнены `anchor` и `title`;
- `anchor` не конфликтует с существующими якорями в разделе;
- ответ самодостаточный и читается без контекста соседних вопросов;
- дублирующийся контент вынесен в ссылку на существующий `anchor`.

## Антипаттерны

- объединять в одном блоке несколько разных проблем;
- использовать общие `anchor` вида `test`, `new`, `example`;
- заполнять `tag` повторением слов из `title` без новых поисковых формулировок.
