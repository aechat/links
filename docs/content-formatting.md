# Правила форматирования текстового контента статей

Настоящие правила определяют требования к форматированию и разметке текстового контента во всех категориях статей, расположенных по пути `src/pages/sections/*`. Соблюдение этих правил необходимо для обеспечения стилистического единообразия, удобства чтения материалов на различных устройствах, включая мобильные экраны, а также для корректной работы поискового индексатора.

## Оформление абзацев

Абзацы выполняют роль вводных элементов, которые вводят читателя в курс дела, объясняют ключевые причины возникновения ошибок и формулируют итоговые выводы. Для сохранения динамики чтения абзацы должны оставаться короткими, обычно в пределах трех или четырех предложений, причем каждый отдельный абзац должен нести строго одну законченную мысль.

В начале любого ответа обязательно размещается вводный абзац, указывающий на вероятную причину возникновения проблемы. В нем рекомендуется использовать формулировки вида «Скорее всего...» или «Обычно это связано с...».

```tsx
<p>
  Скорее всего, вы пытаетесь перезаписать файл, который сейчас используется другой
  программой, например открыт в видеоплеере или отправляется в{" "}
  <mark className="app">Telegram</mark>.
</p>
<p>
  Для решения проблемы закройте программу, которая использует файл, и попробуйте
  снова. Если ошибка остаётся, перезагрузите устройство.
</p>
```

## Оформление списков

Списки используются в ответах для последовательного перечисления шагов решения проблемы или альтернативных способов её устранения. Каждый отдельный элемент списка, представленный тегом `li`, должен содержать законченное предложение, описывающее конкретное действие.

Чтобы не затруднять чтение контента на мобильных экранах, следует избегать глубокой вложенности списков, ограничивая её максимум двумя уровнями. При необходимости внутри пунктов списков допускается размещение блоков дополнительных пояснений и уточнений с использованием компонента `Addition`.

```tsx
<ul>
  <li>
    Попробуйте экспортировать вашу секвенцию в другой формат видео, например в{" "}
    <mark className="video">QUICKTIME</mark> с кодеком{" "}
    <mark className="video">Apple ProRes 422</mark> или использовать сторонние плагины для
    экспорта.
  </li>
  <li>
    Перед экспортом рекомендуется очистить медиакэш. Для этого перейдите в{" "}
    <mark className="select">«Edit» → «Preferences» → «Media Cache»</mark> и нажмите
    кнопку <mark className="select">«Delete»</mark>.
  </li>
</ul>
```

## Разметка таблиц

Таблицы применяются для структурирования информации, сравнения настроек, перечисления технических параметров и двуязычного перевода названий элементов интерфейса. При верстке таблиц обязательно должна соблюдаться правильная структура с использованием валидных семантических тегов, к которым относятся `<table>`, `<thead>`, `<tbody>`, `<tr>`, `<th>` и `<td>`.

Для корректного форматирования названий программ, эффектов или плагинов внутри ячеек применяются соответствующие классы элемента `<mark>`, такие как `app` или `plugin`. Если в ячейках таблицы размещаются внешние ссылки, для них рекомендуется задавать атрибут `title`, который обеспечивает лучшую доступность и повышает удобство чтения контента программами экранного доступа.

```tsx
<table>
  <thead>
    <tr>
      <th>Эффект на русском</th>
      <th>Эффект на английском</th>
      <th className="where-find-index-column">
        Описание эффекта
        <br />
        на <mark className="web">YouTube</mark>
      </th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <mark className="plugin">Извлечение канала 3D</mark>
      </td>
      <td>
        <mark className="plugin">3D Channel Extract</mark>
      </td>
      <td>
        <a
          href="https://youtu.be/qAKdcIeku3Q&t=6"
          title="Видео от Jake In Motion"
        >
          Видео
        </a>
      </td>
    </tr>
  </tbody>
</table>
```

## Логическое разделение с помощью Divider

Компонент `Divider` из библиотеки `antd` используется для логического разделения подразделов или шагов внутри одного раскрытого спойлера. Для смыслового деления контента данный компонент применяется в тех случаях, когда ответ содержит несколько последовательных этапов, таких как диагностика и последующее пошаговое исправление.

Текст разделителя пишется с большой буквы и начинается с глагола в настоящем времени, например, «Проверяем...», «Настраиваем...» или «Экспортируем...». Если внутри заголовка разделителя упоминается программа или плагин, их необходимо оборачивать в соответствующие классы разметки, используя класс `mark` с атрибутами `className="app"` или `className="plugin"`. Для правильного использования компонента его необходимо предварительно импортировать в файл.

```tsx
import {Divider} from "antd";

<Divider>
  Экспортируем <mark className="video">H.264</mark> с нечётным разрешением
</Divider>;
```

> [!WARNING] Не следует перегружать текст разделителями. Если в ответе требуется более двух компонентов `Divider`, рекомендуется рассмотреть возможность разбиения его на отдельные компоненты `DetailsSummary` или использования `NestedDetailsSummary`.

## Оформление версий программ и аппаратных платформ

При упоминании релизного цикла программ используется формат, состоящий из названия программы, мажорной версии и года в круглых скобках, например, `Название программы major.X (YYYY)`. Порог совместимости записывается в виде `... major.X (YYYY) и выше`, при этом для процессоров и аппаратных линеек применяется формулировка `и новее`.

Точная версия программы фиксируется в виде `... x.y[.z] (YYYY)`. Типовыми примерами такой записи являются `Adobe After Effects 23.X (2023)`, `Adobe Premiere Pro 25.X (2025)` и `Adobe Photoshop 24.X (2023)`.

## Выделение элементов с помощью mark и семантических классов

Стили компонента находятся в файле `src/styles/components/_mark.scss`. Тег `mark` применяется для выделения сущностей, которые проще воспринимаются с визуальным маркером, что обеспечивает единый стиль оформления для файлов, путей, приложений, кнопок и терминов.

Класс выбирается строго по смыслу, а не ради красивой иконки. Длинные абзацы из сплошных выделений `mark` не допускаются, для обычного текста всегда должен оставаться тег `p`.

Для структурирования контента и визуального выделения различных элементов в статьях применяется тег `<mark>` со специализированными CSS-классами. Класс `app` используется для названий программ и приложений, таких как `<mark className="app">Adobe After Effects</mark>` или `<mark className="app">Shutter Encoder</mark>`. При этом для названий приложений `mark.app` используется только в контенте статьи. Вставлять `mark` в `title`, `caption` или в текст ссылок `<a>…</a>` нельзя.

Для обозначения номеров версий и релизов предназначен класс `version`, например, `<mark className="version">23.6 (2023)</mark>`. Названия плагинов и расширений размечаются классом `plugin`, к которым относятся `<mark className="plugin">Voukoder</mark>` или `<mark className="plugin">AfterCodecs</mark>`. Класс `video` служит для выделения видеокодеков, форматов сжатия и контейнеров, таких как `<mark className="video">H.264</mark>` или `<mark className="video">Apple ProRes 422</mark>`. Смежные классы `image` и `audio` используются для обозначения соответствующих типов медиафайлов.

Технические термины, протоколы и стандарты размечаются с помощью класса `term`, например, `<mark className="term">ASCII</mark>` или `<mark className="term">GPU-ускорение</mark>`. Пункты меню, пути в интерфейсе программы и строки системных предупреждений оборачиваются в класс `select`, к примеру, `<mark className="select">«Help» → «About After Effects»</mark>` или `<mark className="select">«Output file will be resized»</mark>`. Названия элементов интерфейса, такие как поля, кнопки, пункты меню и параметры, оформляются через `mark.select` обязательно с использованием кавычек-ёлочек `« »`. Для клавиш и клавиатурных сочетаний предназначен класс `key`, например, `<mark className="key">Ctrl + K</mark>`.

Названия компаний-разработчиков выделяются классом `company`, например, `<mark className="company">Adobe</mark>`. Для файловой системы доступны классы `file` и `path`. Класс `file` используется для форматов файлов, их расширений и системных путей, в частности, `<mark className="file">ae_force_english.txt</mark>` или `<mark className="file">HOSTS</mark>`. Класс `path` предназначен для путей в файловой системе.

Для параметров, ключей реестра и значений, предназначенных для копирования кликом, используется класс `copy`, например, `<mark className="copy">ApplicationLanguage</mark>` или `<mark className="copy">en_US</mark>`. Числовые значения, которые вводятся вручную, также оформляются через `mark.copy` без кавычек. Если единица измерения или суффикс задаётся самим интерфейсом, например `Кбит/с`, `МБ`, `%`, она выносится за пределы тега `mark.copy`.

Дополнительно в разметке могут использоваться класс `user` для пользователей, `web` для веб-ресурсов, а также классы `tag`, `word`, `quote`, `code`, `file-code`. Для аппаратных компонентов предназначен класс `hardware`, для параметров кодирования используется `codec-param`, а для цветовых пространств – `color-space`. Статусные метки оформляются с использованием классов `info`, `warning` и `danger`.

Отображаемые версии, релизы и диапазоны в тексте оформляются через `mark.version`. Диапазон версий с названием программы записывается в виде «от … до …» с классом `mark.app`. Версии и диапазоны без названия программы пишутся обычным текстом, например `23.6 (2023)` или `от 23.1 до 23.5 (2023)`.

Для классов `app` и `plugin` в `mark` оборачивается только название сущности, а версия рядом оформляется через `mark.version`. Для классов `term`, `web` и `company` при конструкции «название + версия» применяется тот же подход, то есть в тег `mark` попадает только название, а версия остается снаружи.

В таблицах перевода и каталогах эффектов, например в файлах `*WhereFind.tsx`, устоявшиеся названия наподобие `Keylight (1.2)` и `CC Light Burst 2.5` не дробятся без отдельной задачи. Устоявшиеся токены и плейсхолдеры, такие как `CC0` и `20XX`, допускается оставлять целиком в теге `mark`.

```tsx
<p>
  Откройте <mark className="path">Edit -> Preferences -> Media & Disk Cache</mark> и
  очистите <mark className="file">Disk Cache</mark>.
</p>
<p>
  Для выражения используйте <mark className="code">wiggle(3, 20)</mark>.
</p>
<p>
  Ресурс: <mark className="web">Envato Elements</mark>.
</p>
<p>
  Параметр кодирования: <mark className="codec-param">CBR</mark>.
</p>
<p>
  Язык: <mark className="file-code">JavaScript</mark>.
</p>
<p>
  Компонент: <mark className="hardware">Intel Core</mark>.
</p>
```

## Компактные наборы ссылок с классом flexible-links

Стили данного компонента находятся в файле `src/styles/components/_flexible-links.scss`. Контейнер с классом `flexible-links` применяется для оформления подборок однотипных ссылок по теме в виде компактного списка ресурсов, визуально похожего на кнопки. Внутри контейнера с классом `flexible-links` размещаются элементы `a` или `button`.

Подписи ссылок должны объяснять, что именно откроется по переходу. Слова «сюда», «тут» и «линк» не используются. Длинные URL не вставляются в текст ссылки, а остаются в атрибуте `href`.

```tsx
<div className="flexible-links">
  <a href="https://example.com/course">Курс для новичков</a>
  <a href="https://example.com/channel">Полезный канал</a>
</div>
```

Элементы отображаются как единая горизонтальная группа кнопок. Внутри раскрывающегося спойлера `DetailsSummary` поддерживается долгое нажатие по ссылке для копирования URL. Поисковый движок `SearchEngine` индексирует ссылки из `.flexible-links a` как отдельный блок.
