# Настройка редактора и запуск проекта

Проект базируется на менеджере пакетов Yarn версии 4. Для установки всех зависимостей перед началом работы в терминале выполняется команда:

```shell
yarn install
```

При первом запуске проекта на новом устройстве требуется настроить Git Large File Storage для скачивания больших бинарных файлов и медиаресурсов. Для этого последовательно выполняются команды:

```shell
git lfs install
git lfs pull
```

Если утилита `git lfs` не распознается терминалом, ее необходимо предварительно установить в систему через установщик или пакетный менеджер вашей операционной системы, после чего повторить ввод команд.

## Запуск разработки

Запуск сервера локальной разработки для тестирования интерфейса осуществляется с помощью команды:

```shell
yarn dev
```

## Проверки и форматирование

Для проверки типов и автоматического форматирования кода применяется команда:

```shell
yarn format
```

Эта команда последовательно запускает компилятор TypeScript `tsc`, инструмент автоматического исправления стилей `stylelint --fix`, линтер кода `eslint --fix` и утилиту форматирования `prettier --write`.

## Поддержка VS Code и Yarn PnP

Для настройки корректного взаимодействия среды разработки VS Code с механизмом Yarn Plug'n'Play выполняется команда интеграции SDK:

```shell
yarn dlx @yarnpkg/sdks vscode
```

## Запуск в Docker

Для развертывания проекта в изолированном контейнере используется стандартная команда запуска Docker Compose:

```shell
docker-compose up
```
